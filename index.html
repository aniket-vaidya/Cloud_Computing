Cloud Computing – Detailed Revision Guide

Unit 1 – Introduction

Introduction to Cloud Computing

Cloud computing is the delivery of computing services—including servers, storage, databases, networking, software, analytics, and intelligence—over the Internet (“the cloud”) to offer faster innovation, flexible resources, and economies of scale.

Features of Cloud Computing

  • On-demand self-service: Users can provision resources automatically without human interaction.
  • Broad network access: Services are available over the network and accessed through standard devices.
  • Resource pooling: Providers serve multiple customers using multi-tenant models.
  • Rapid elasticity: Capabilities can be elastically provisioned and released to scale rapidly.
  • Measured service: Cloud systems automatically control and optimize resource use by metering.

Advantages and Disadvantages of Cloud Computing

Advantages:

  • Cost savings: No need to invest in hardware.
  • Scalability: Easily scale resources up or down.
  • Accessibility: Access data/services anywhere.
  • Maintenance-free: Managed by the cloud provider.

Disadvantages:

  • Security concerns: Data privacy and control.
  • Dependence on internet connectivity.
  • Potential downtime or service outages.
  • Compliance challenges with regulations.

Challenges and Risks of Cloud Computing

  • Data breaches and loss.
  • Insider threats and cyber-attacks.
  • Legal and compliance issues.
  • Vendor lock-in limiting portability.
  • Unpredictable costs from scaling.

Roots of Cloud Computing

Cloud computing has evolved from:

  • Grid computing: Using a distributed network of computers to achieve a common goal.
  • Utility computing: Computing resources billed like utilities (electricity, water).
  • Virtualization: Creating virtual versions of hardware resources.
  • Service-Oriented Architecture (SOA): Building software as reusable services.

Types of Cloud Computing

  • Public Cloud: Services offered over the public internet, shared among users (e.g., AWS, Azure).
  • Private Cloud: Cloud infrastructure operated solely for a single organization.
  • Hybrid Cloud: Combination of public and private clouds to allow data and apps sharing.
  • Community Cloud: Shared cloud infrastructure for a specific community with common concerns.
  • Multi-Cloud: Using multiple cloud services from different providers.

Unit 2 – Cloud Architecture, Services and Applications

Cloud Computing Stack

The cloud stack is generally divided into layers of services:

  • Infrastructure as a Service (IaaS): Provides virtualized computing resources like servers, storage, and networks.
  • Platform as a Service (PaaS): Provides platforms allowing customers to develop, run, and manage applications.
  • Software as a Service (SaaS): Software applications delivered over the internet, eliminating installation.
  • Identity as a Service (IDaaS): Cloud-based identity and access management solutions.

Infrastructure as a Service (IaaS)

Allows users to rent IT infrastructure like servers and storage on a pay-as-you-go basis.

Platform as a Service (PaaS)

Provides a development platform and tools for building applications without managing underlying infrastructure.

Software as a Service (SaaS)

Delivers software applications over the internet on a subscription basis.

Identity as a Service (IDaaS)

Cloud-based identity management services enabling authentication and authorization.

SaaS vs PaaS

  • SaaS: Complete software delivered to end-users.
  • PaaS: Platform enabling developers to build and deploy applications.

Unit 3 – Abstraction and Virtualization

Introduction to Virtualization

Virtualization is the process of creating virtual versions of physical resources such as servers, storage devices, or networks, allowing multiple operating systems to run on a single physical machine.

Types of Virtualization

  • Server Virtualization: Running multiple virtual servers on a single physical server.
  • Desktop Virtualization: Running desktop environments remotely or in virtual machines.
  • Network Virtualization: Combining hardware and software network resources.
  • Storage Virtualization: Pooling physical storage from multiple devices into a single storage resource.

Load Balancing Techniques

Load balancing distributes workloads across multiple resources to optimize resource use, maximize throughput, reduce latency, and ensure fault tolerance. Common techniques:

  • Round Robin
  • Least Connections
  • IP Hash
  • Weighted Load Balancing

Introduction to Hypervisor

A hypervisor is a software or firmware layer that creates and manages virtual machines (VMs) by abstracting hardware resources. There are two main types:

  • Type 1 (Bare-metal): Runs directly on the physical hardware (e.g., VMware ESXi, Microsoft Hyper-V).
  • Type 2 (Hosted): Runs on top of an existing operating system (e.g., Oracle VirtualBox, VMware Workstation).

Storage Area Network (SAN)

SAN is a dedicated high-speed network that provides block-level storage access to servers. It enables multiple servers to access a centralized pool of storage devices, improving performance and scalability.
